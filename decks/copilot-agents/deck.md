# GitHub Copilot で動かす「自作エージェント」

目的別の AI スタッフを用意し、合図ひとつで仕事を任せる仕組み

<small>※社内機密データは含まない、構成の概要共有用</small>

---

## エージェントとは？

ふつうの AI チャットは「1回の質問 → 1回の回答」。
**エージェント**は、そこに 3 つを足したもの。

- **役割** … 例「リサーチ係」「会議メモ係」
- **手順** … 決まった段取り（テンプレ・保存先まで指定）
- **トリガー（合図）** … 特定の言葉を言うと自動で起動

> = 目的別の専門スタッフを何人も置き、合図で呼び出せる

---

## 大きく 2 タイプ

| タイプ | イメージ | 例 |
|---|---|---|
| **単体エージェント** | 1人の専門スタッフ | リサーチ係、会議メモ係 |
| **チーム型（オーケストレーター）** | 監督が複数に分業指示 | 記事を執筆→事実確認→読みやすさ確認で回す |

---

## 単体エージェント（目的別スタッフ）

| エージェント | 何をする係か | 合図の例 |
|---|---|---|
| リサーチ | Web を多角的に調べ出典付きで整理 | 「リサーチして」 |
| 会議アクション整理 | 議事録＋アクション抽出、ToDo登録 | 「会議メモを整理して」 |
| AI回答ドラフト | 顧客質問への返信案を作成（未送信） | 「返信ドラフト作って」 |
| レジュメ／キャリア | 経歴整理・企業調査・面接準備 | 「レジュメ」「転職」 |
| 月次／事業レビュー（社内） | 音声メモを経営層向け英語資料に | 「MBR作って」 |
| セッション振り返り | 改善点を自動で提案 | 「ありがとう」 |

---

## チーム型エージェント（分業の流れ）

監督役が複数の専門係に分担。**チェックを別の係に回すと品質が安定**。

- **技術記事（Zenn）**：監督 → 執筆 → 事実確認 → 読みやすさ確認
- **SNS投稿（M365）**：監督 → リサーチ → 執筆 → 批評
- **機器開発（ファームウェア）**：監督 → 実装 ⇄ 批評（改善サイクル）
- **ネット通販（温泉EC）**：監督 → マーケ／戦略／実務（並列）

## 作り方①：登場するファイルは 3 種類

| 種類 | 置き場所 | 役割 |
|---|---|---|
| エージェント定義 | `.github/agents/◯◯.agent.md` | 合図と役割を書く「名札＋マニュアル」 |
| スキル（手順書） | `skills/◯◯.md` | 細かい段取り・テンプレ |
| ルーティング表 | `copilot-instructions.md` | どの合図でどの係を呼ぶか一覧 |

> 定義ファイルは軽い受付係、詳しい手順はスキルに分けるのがコツ

---

## 作り方②：エージェントは「MD 1枚」

`---` で囲む **設定欄（フロントマター）** ＋ **指示書（本文）** の 2 部構成。

```markdown
---
name: mbr                     ← 名前
description: |
  Trigger phrases:            ← ★この言葉で起動
  - 'MBR作って'
---
You are an MBR assistant ...   ← 役割
Workflow:                      ← 手順
1. 入力を読む 2. 生成 3. 保存
Output: 箇条書きで / 保存先を指定
```

特別な開発は不要。テキスト 1 枚でスタッフが 1 人増える。

---

## 作り方③：合図で起動する流れ

1. 利用者が合図（例「リサーチして」）を入力
2. ルーティング表で合図を照合
3. 一致した定義（.agent.md）を読み、その役になりきる
4. 必要ならスキル（手順書）と資料を参照
5. 決められた場所に成果物を保存

> 現状：エージェント定義 約15個 ／ 手順書（スキル）約27個

---

## 実際に任せている仕事

- **調べもの** … 市場調査・最新動向リサーチ（出典付き）
- **文章づくり** … 記事・SNS投稿・資料下書き・返信案
- **会議まわり** … 議事録・アクション抽出・ToDo登録
- **キャリア支援** … レジュメ最適化・企業調査・面接準備
- **モノづくり** … 機器ファームウェアの実装とレビュー
- **自己改善** … 振り返りから次の改善点を提案

---

## 効いている工夫

- **合図ひとつ**で呼び出せる（毎回の指示文が不要）
- **分業**でミスを減らす（書く係 × チェック係）
- **出力先まで自動化**（保存場所・形式を固定）
- **送信は必ず人間**（AIは下書きまで＝安全の鉄則）
- **振り返りで育てる**（繰り返し作業を自動化へ昇格）
